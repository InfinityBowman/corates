# Bug hunt findings (2026-07-14)

Inventory from the test-first bug hunt. Every item is pinned by a test; unfixed
items are marked with it.fails (unit) or test.fixme (e2e) so CI stays green
while the bug is documented. When a bug gets fixed, vitest flips its it.fails
test to failing, forcing the marker's removal. Bug numbers are stable
identifiers referenced by the test files. Status reflects the bughunt branch
(PR #529) as of 2026-07-21.

## Fixed on this branch

**Bug 2. Concurrent first-writes destroy one reviewer's data.**
Container Y.Maps (a study's annotations, pdfs, reconciliations, per-checklist
annotation sub-maps, and meta.outcomes) were lazily created on first write. Two
clients making their first write concurrently (live or one offline) each
created their own container; Yjs keeps one and permanently discards the other's
contents on every client. Dual review makes "two reviewers on a fresh study"
the normal case. Fixed by pre-creating containers with their parents
(`studies.ts`, `checklists/index.ts`), flat per-field reconciliation keys
(`reconciliation.ts`), and the server-side ensureDocContainers backfill for
existing docs (which writes under a reserved low clientID so its empty
containers lose any conflict with a client's populated one). Tests:
`web/src/primitives/__tests__/bughunt-concurrent-container-creation.test.ts`,
`workers/src/durable-objects/__tests__/ensure-containers.test.ts`,
`e2e/bughunt-outcome-first-write-race.spec.ts`.

**Bug 3. Users stranded when access is revoked or a project is deleted.**
Two defects: the DO rejected the WebSocket upgrade over HTTP (403/404), which a
browser cannot distinguish from an unreachable server; and access-denied
cleanup reset the connection state in the same tick as the ACCESS_DENIED
dispatch, so ProjectGate's redirect effect never observed the error. Denial is
now delivered as accept-then-close with code 1008 and an explicit reason, and
the gate owns toast, redirect, then cleanup. Tests:
`e2e/bughunt-member-removal.spec.ts`, `e2e/bughunt-project-delete.spec.ts`,
`workers/src/durable-objects/__tests__/ProjectDoc.ws-auth.test.ts`.

**Bug 18, interim fix. Connection failures now fail loudly.**
The 5-error threshold no longer guesses "may have been deleted" and no longer
wipes local Dexie data (including unsynced edits) on plain outages; ProjectGate
shows an explicit connection-trouble state with a retry action after 15s
instead of an infinite skeleton. Member offline cache itself is still missing
(bug 18 below).

## High severity, deferred

**Bug 0. Invoice status resurrection after cancellation.**
handleInvoicePaymentSucceeded
(`packages/stripe-purchases/src/routes/handlers/invoiceHandlers.ts:78-85`)
looks up the row by subscription id and unconditionally sets status: 'active'
without checking existing.status. Stripe does not guarantee event ordering and
retries webhook deliveries for up to ~3 days, so a late or retried
invoice.payment_succeeded arriving after cancellation flips a canceled
subscription back to active, and nothing ever corrects it because a dead
subscription emits no further events. Fix: skip the update when
existing.status === 'canceled' (or endedAt is set), or derive status from a
canonical Stripe re-fetch like syncStripeSubscription does.

**Bug 1. Transactional emails silently dropped after one transient failure.**
`packages/workers/src/queue.ts:28` records the queue message id in
processed_emails BEFORE attempting the send. If Postmark fails once, the
redelivery sees "already processed" and acks without sending. OTPs,
invitations, and contact emails are lost on a single network blip; the
retry/backoff config is dead code. Test:
`workers/src/lib/__tests__/bughunt-email-queue-idempotency.test.ts`.

**Bug 4. ROBINS-I Domain 2 dead-ends on valid answers.**
`shared/src/checklists/robins-i/scoring-domains.ts:383-409`: on the
not-distinguishable branch, 2.4 = "Weak Yes" or "No Information" (both
clickable options) yields no judgement; 480 fully-answered legal combinations
can never complete the checklist. Test:
`shared/src/checklists/__tests__/bughunt-robins-scoring-deadends.test.ts`.

**Bug 5. Reviewer-2-only studies strand completed appraisals.**
`shared/src/checklists/domain.ts:244`: the single-reviewer check only
recognizes slot 1, so { reviewer1: null, reviewer2: userX } (which the assign
modal saves without complaint) completes into "awaiting reconciliation"
forever: locked, and invisible in all three workflow tabs. Test:
`web/src/lib/__tests__/bughunt-reviewer2-only-study.test.ts`.

**Bug 6. RIS import leaks each record's last field into the next.**
`web/src/lib/referenceParser.ts:73-95`: currentTag is not reset at record
boundaries, so multi-record Zotero/EndNote uploads get contaminated
abstracts/DOIs and phantom first authors. Test:
`web/src/lib/__tests__/bughunt-referenceParser.test.ts`.

**Bug 17. Multi-outcome checklist-add dead end.**
The checklist add form only exists inside a study's To Do row, and
shouldShowInTab('todo') (`shared/src/checklists/domain.ts:111-118`) hides the
study once every checklist assigned to the current user is reviewer-completed.
A reviewer who completes their outcome-A checklist before adding outcome-B's
loses the study from To Do permanently; no UI path remains to start the second
appraisal. Deferred pending the checklist-add flow redesign. Test:
`e2e/bughunt-multi-outcome-add-deadend.spec.ts` (fixme).

**Bug 18. Members have no working offline cache.**
ConnectionPool.initializeConnection (`ConnectionPool.ts:175-196`) registers the
Dexie write-back handler after the initial WebSocket state was already applied,
so a member's IndexedDB never holds the base project state; only the owner
renders from cache. Full member offline support is deferred (decision
2026-07-20); the interim fail-loudly state above covers the
reload-while-unreachable case. Test:
`e2e/bughunt-member-offline-cache.spec.ts` (fixme).

## Medium severity, deferred

**Bug 7. ROBINS-I Domain 4 never scores when 4.11 = NA.**
`shared/src/checklists/robins-i/scoring-domains.ts:608-728`: 4.11 offers NA in
the UI, normalizeAnswer maps NA to NI, but every 4.11 check handles only Y/PY
and N/PN, so the domain is stuck; ~162k combinations dead-end on the
missing-data path. Test:
`shared/src/checklists/__tests__/bughunt-robins-scoring-deadends.test.ts`.

**Bug 8. Stale PDF download replaces the open preview.**
`web/src/stores/pdfPreviewStore.ts:71` plus `web/src/project/actions/pdfs.ts`:
neither closePreview() nor a later openPreview() cancels an in-flight download,
and setData()/setError() carry no identity of which view() they belong to.
Close during a slow download of PDF A, open PDF B, and A's late bytes render
under B's title; a late failure for A replaces B's rendered document with an
error panel. Test: `web/src/stores/bughunt-pdf-preview-race.test.ts`.

**Bug 9. Annotation watchers never see annotations added after the container
is created.** `web/src/primitives/useProject/useStudyAnnotations.ts:74-116`:
when the hook mounts before study.annotations exists, it attaches a shallow
fallback observer to the study map and never upgrades it to a deep observer on
the created annotations map, so a second reviewer with the PDF open sees
nothing new until they reopen the view. Test:
`web/src/primitives/__tests__/bughunt-stale-annotation-subscription.test.ts`.

**Bug 10. Opening a project offline permanently wedges the connection.**
`web/src/primitives/useProject/connection.ts`: connect() while
navigator.onLine is false sets shouldBeConnected but creates no provider, and
both recovery paths (the manager's own 'online' handler and ProjectGate's
reconnectIfNeeded, which gates on getShouldReconnect()) require a provider to
exist. Edits accumulate unsynced until the user leaves and re-enters the
project. Same class as the Harly incident. Test:
`web/src/primitives/__tests__/bughunt-offline-connect-wedge.test.ts`.

**Bug 11. Notifications lost on half-dead sockets.**
`workers/src/durable-objects/UserSession.ts:104`: notify() calls ws.send()
bare, with no try/catch. A socket that looks OPEN but whose connection died
(laptop sleep, wifi drop) makes send() throw, skipping the user's other tabs
and the pendingNotifications fallback queue, so the notification is permanently
lost instead of delivered on next connect. Test:
`workers/src/durable-objects/__tests__/bughunt-usersession-notify.test.ts`.

**Bug 12. BibTeX pages = {100--105} parses to null.**
`web/src/lib/referenceParser.ts:232`: the standard double-hyphen page range
used by Zotero/BibTeX exports never matches the parser's pattern. Test:
`web/src/lib/__tests__/bughunt-referenceParser.test.ts`.

**Bug 13. BibTeX brace-protected titles truncate.**
`web/src/lib/referenceParser.ts`: a title like {The {COVID-19} Pandemic}
imports as "The" because nested case-protecting braces end the field value
early. Test: `web/src/lib/__tests__/bughunt-referenceParser.test.ts`.

## Low severity, deferred

**Bug 14. AMSTAR2 q9/q11 half-answered rating depends on which half was
answered.** `shared/src/checklists/amstar2/answers.ts:71`: consolidateAnswers
merges the two sub-questions by keeping the q9a object, so q9a=Yes with q9b
unanswered scores differently from q9a unanswered with q9b=Yes even though
they carry identical information. Reachable via export of in-progress local
appraisals, which scores without a completeness gate. Test:
`shared/src/checklists/__tests__/bughunt-amstar2-q9-consolidation.test.ts`.

**Bug 15. Email subjects glue words together.**
`shared/src/html.ts` sanitizeEmailSubject deletes control characters (tab, CR,
LF) BEFORE the whitespace collapse, so words separated only by a tab or
newline are concatenated. A tab pasted into a project name survives to the
server. Test: `shared/src/__tests__/bughunt-html.test.ts`.

**Bug 16. Subject truncation can split an emoji surrogate pair.**
The 78-character truncation in sanitizeEmailSubject can cut an emoji in half,
leaving a lone surrogate (mojibake) in the subject line. Test:
`shared/src/__tests__/bughunt-html.test.ts`.
